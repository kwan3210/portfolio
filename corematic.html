About

Delievered as part of the Human-Computer Interaction course at UQ, the project brief required us to work through an interaction design and human-centred design process for an interactive system. The goal for the project was learning about HCI, UX and usability and not producing a polished product.

The Challenge

Design and develop an application to be used on a mobile device that tracks and displays the expenses of an individual and provides feedback to help them reduce unnecessary expenses, work towards user set saving goals and ultimately better manage their budgets.



Design Process

  • length: 3 months, in studio
  • role: User testing, UX/UI research + design, Prototyping
  • phases: research + design + prototype
  • platforms: mobile


    • New Dashboard UI

    At this point, Savr is deep into the agile design process, with this most recent iteration of the Savr UI interface completing the fourth cycle of the IDCL, short for Interaction Design Lifecycle (Hartson and Pyla, 2012) as shown in diagram. In order to continue improving on Savr, it is important to move into the evaluation stage once again and test the changes made to Savr, then learn how the changes made to Savr impacted user experience and how further changes can be integrated to improve the prototype.
